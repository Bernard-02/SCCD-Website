## Part 1 — 一個大學系所網站要符合什麼標準？

### 台灣法規脈絡
- 主管標準＝**數位發展部「網站設計無障礙規範」**（前身為 NCC / 研考會的「網站無障礙規範」），技術基礎對齊國際 **WCAG 2.1**。
- 分三等級：**A（最低）／ AA（建議）／ AAA（最高）**。
- 現制為**「數位無障礙自我宣告」**（取代舊的無障礙標章審驗）：網站自評後在數位發展部平台宣告符合等級。
- 法律強制對象主要是**政府／公部門網站**。實踐大學為**私立大學**，法規上非強制；但：
  - 教育機構普遍對齊政府規範（招生公平、形象、補助申請常被要求）。
  - 《身心障礙者權益保障法》精神 + 校務評鑑常涵蓋數位近用。
- **務實目標：以 WCAG 2.1 AA 為基準**（等同台灣規範 AA），這也是國際業界共識門檻。

### WCAG 四大原則（POUR）
| 原則 | 意義 | 本站最相關的點 |
|---|---|---|
| **可感知 Perceivable** | 資訊要能被看到/聽到 | 圖片 alt、色彩對比、文字可縮放 |
| **可操作 Operable** | 全功能要能不靠滑鼠操作 | 鍵盤操作、焦點可見、skip link、可暫停動畫 |
| **可理解 Understandable** | 內容與操作可預期 | lang、標題階層、表單標籤、一致導航 |
| **強健 Robust** | 輔助科技能正確解析 | 語義 HTML、ARIA、SPA 換頁通知 |

### 🔧 Tier 5 — 標題階層（1.3.1 A）：根本做法 = tag/size 解耦
> **政策**：標題的 **tag = 語意層級**（一頁一 h1 → h2 分區 → h3 子項）；**視覺大小 = `text-hN` class**。
> 機制：typography.css 的 `h1{@apply text-h1}`…`h6{}` 是預設（**不動**）；顯式 `text-hN` class（特異度 0,1,0）贏過 tag（0,0,1）→ `<h2 class="text-h1">` = 語意 h2、視覺 h1。**全程不用 inline、不動 CSS/變數**。三型修法：①分區標題降級保大字 `<h1>`→`<h2 class="text-h1">`；②內文誤當標題 `<h5>`→`<p class="text-h5 font-bold…">`（補回 tag 原本給的 font-bold）；③頁標題升級 `<h2 class="hero-title">`→`<h1 class="hero-title text-h2">`。
