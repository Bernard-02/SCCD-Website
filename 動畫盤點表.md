## 架構總覽（手動維護層）

### 1. 兩大進退場 pattern

| Pattern | 機制 | 視覺 | 適用元素 | 共用實作 |
|---|---|---|---|---|
| **clip-reveal**（=「y reveal」=「hero title 那個」）| `yPercent: 0 ↔ ±100` + 外層 `overflow:clip` wrapper | 在容器邊緣乾淨滑入 / 沉出 | 流式 block：文字段落 / list row / hero 文字 / 描述塊 | `scroll-animate.js` `setupClipReveal` / `playClipReveal` / `playRevealExit` |
| **clip-path** | `clip-path: inset(...)` 四方向 100%↔0% wipe | 擦除 / 抹掉感 | 不重組 DOM 的方塊：圖片 / 卡片 / nav chip / header bar / logo | `scroll-animate.js` `playClipPathExit`；`lightbox-shell.js` `animateHeaderHide` / `animateHeaderShow` |

**選用規則**：流式 block → clip-reveal；grid/flex 卡片、圖片、chip、chrome → clip-path。
**為何分**：clip-reveal 動態 wrap 會 reparent 破壞 `parent > child` 選擇器、且不適合 grid/flex item；clip-path 不改 DOM、安全給卡片，但起點 `none` 補間會 snap（見陷阱 ①）。

### 5. 方向 / timing 慣例

- **ease 方向**：進場/reveal → `power*.out`；退場/hide → `power*.in`；雙向 layout → `power2.inOut`；scrub → `none`；marquee → `linear` + `repeat:-1`。
- **招牌 clip-reveal**：`DUR.reveal`(1.0s) + power3.out + stagger 0.12（同層）/ 0.08（跨卡）；嚴格依序進場（title→副標→內文）傳線性 `{each:0.12}`（grid-auto 在非均勻 layout 會亂猜順序）。
- **CSS 兩主力曲線**：clip-path 擦除 `cubic-bezier(0.25, 0, 0, 1)`；Material 通用 `cubic-bezier(0.4, 0, 0.2, 1)`。
- **mode 切換 fade**：0.4s（whitelist 在 `typography.css`，新增 mode-aware 元件要補進 whitelist）。

### 6. 反覆踩過的陷阱（改動畫前自檢）

1. **clip-path 退場起點 = none**（前面 reveal 用過 `clearProps:'clipPath'`）→ 必用 `gsap.fromTo` 顯式 `inset(0)`；`gsap.to` 從 none 補不動會 snap。
2. **元素帶 CSS `transition:all`**（含 Tailwind `transition-all`）→ GSAP 每幀寫 clip-path/transform 被 CSS 追著 = 漸進落後再 snap → 動畫前 `el.style.transition='none'`。
3. **clip-reveal 的 wrapper 必須 ≈ 內容高才藏得住**：grid/flex item 被 align stretch 撐高 → `yPercent:100` 藏不住先露 → wrapper `align-self:start`。
4. **量文字寬高排版/避碰前先 `await document.fonts.ready`**（fallback 字型低估寬度 → 字型 swap 後撐破邊界）。
5. **ambient（RAF 每幀寫某屬性）元素**的進退場要動「RAF 沒在寫的屬性」否則每幀被覆蓋打架（floating 卡 RAF 寫外層 transform → clip-path 套內層卡片）。
6. **容器被 scrub ScrollTrigger 佔了 clipPath** → 退場 clip 子元素自身、別碰容器。
7. **改 SPA 內 `js/*.js` 要 hard reload**（module 常駐跑記憶體舊版）；改被 `input.css` @import 的 component CSS（legal/lists/footer/...）要 `npm run build:css`。

### 7. Footer 動畫架構（持久元素特例）

footer 不被 router swap（只 init 一次），**不能用 `registerPageExit`**；退場/復位由 router 在換頁流程直接驅動：
- `runPageExit` 階段呼叫 `playFooterExit()`（footer 在視窗內才跑、回 Promise 讓 router await）；
- swap + scrollToTop 後呼叫 `resetFooterAfterExit()`（此時 footer 已捲離 → 重進場不被看到、純復位）。
- **viewportOnly**：捲在上方點 header 連結 → footer 不在畫面 → no-op 不打斷 shuffle；`_footerExited` flag gate reset、`_footerExitResolve` 處理連點。

離頁三組元素**同時、同速、時長一致**（`FOOTER_EXIT_DUR`=`DUR.medium`(0.5s) / `EASE.exit`(power3.in) / 無 stagger）＝**直接照 hero exit**（hero-animation.js `EXIT_DURATION=0.5`+`EASE.exit`）：
> ⚠️ 之前用 0.75s 時 power3.in 前段 ~0.45s 幾乎不動＝「delay 很久才走」（user 2026-06-08，中途試 power2.in@0.75 仍怪）→ 真因是**長時長放大 power3.in front-load**、非 ease 本身；縮回 hero 的 0.5s 就順（短時長下 power3.in 前段空白不明顯）。playwright：498ms 收完、250ms 6%→375ms 35%→500ms 99%。

| 組 | 元素 | 退場做法 |
|---|---|---|
| 散佈 items（8）| `.footer-*` | clip-reveal 雙軸隨機四向沉出 |
| 左側 logo | `.footer-logo-area` | clip-path（`animateHeaderHide`，跟 header logo 一致）|
| 規章區 | `.footer-privacy a`（4 連結）| clip-reveal 單軸 yPercent 沉出，**每連結各自 `setupClipReveal` 包 overflow:clip 遮罩、stagger 0 一起出場**（`<a>` 需 `display:block` 才吃 transform）★2026-06-08 由 clip-path 改 |

平常 10s shuffle：items exit（random dir 0.6s）→ onComplete 即時 generate+verify 新 layout（30 次失敗才 fallback）→ reveal；hover 任一 item 凍結倒數、離開續跑。
